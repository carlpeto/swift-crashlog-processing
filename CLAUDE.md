## What This Is

`swift-symbolicate` is a CLI tool and library that adds symbols into backtraces generated by the built-in Swift runtime crash logger (`swift-backtrace`). It reads a stream (stdin or file), detects embedded crash logs in JSON or plain text format, symbolicates them inline, and passes all other input through unchanged. Multiple crash logs in a single stream are supported.

## Commands

```bash
# Build all targets
swift build

# Run all tests
swift test

# Run a single test by name
swift test --filter "testName"

# Run only general (non-symbolication) tests
swift test --traits TestGeneral

# Run only symbolication tests (requires crashMe binary)
swift test --traits TestSymbolicating

# Build and run the tool manually
swift run swift-symbolicate [--symbol-additional-paths <path>] [--output-file <file>] [<input-file>]
```

Behaviour can be tuned at runtime via the `SWIFT_BACKTRACE` environment variable (same as the Swift runtime uses), e.g.:
```bash
SWIFT_BACKTRACE="threads=crashed,demangle=no,sanitize=yes" swift run swift-symbolicate crash.txt
```

## Architecture

### Stream Processing Pipeline

```
InputStream → CrashScanner → process closure → OutputStream
```

`CrashScanner` (`Sources/SwiftSymbolicate/CrashScanner.swift`) reads byte-by-byte from an input stream, uses `Recognizer` state machines to detect crash log start/end boundaries, buffers matched bytes, then calls a processing closure with the captured crash log data. Unmatched bytes are passed directly to the output stream.

### Recognizer State Machine

`Recognizer` (`Sources/SwiftSymbolicate/Recognizer.swift`) is a byte-by-byte pattern matcher. Signatures are composed of `SignaturePart` elements — either `.exact(byte:)` for literal matches or `.skipTo(endByte:maxRun:)` for variable-length sections. Returns `.noMatch`, `.recognizing`, `.failed`, or `.complete`.

### Format Detection

`LogStreamReaderWriter` (`Sources/SwiftSymbolicate/LogStreamReaderWriter.swift`) is the protocol that pairs start/end recognizers for a given log format:
- `JsonLogStreamReaderWriter` — detects `{ "timestamp": "...", "kind": "crashReport"` ... `"backtraceTime": ...}`
- `PlainTextLogStreamReaderWriter` — detects ` Program crashed: ` ... `Backtrace took ...s`

### CrashLog Data Model

The `CrashLog<Address>` generic type is from the `Runtime` module (Swift stdlib internals), accessed via `@_spi` annotations. The tool uses `HostCrashLog = CrashLog<HostContext.Address>` throughout.

### Readers and Writers

- `PlainCrashLogReader<Address>` — state-machine parser for plain text crash logs; populates a `CrashLog` model
- `PlainCrashLogWriter` — serializes `CrashLog` back to plain text
- `exportAsJson` / `InlineBacktraceWriter` (`JSONCrashLogWriter.swift`) — serializes `CrashLog` to JSON using `BacktraceJSONFormatter` from the `Runtime` module

### Symbol Location

`OfflineSymbolLocator` (`Sources/SwiftSymbolicate/OfflineSymbolLocator.swift`) extends `DefaultSymbolLocator` from the `Runtime` module. It searches the image's original path and any `--symbol-additional-paths` for matching binary/symbol files, using the crash log's platform path separator to extract filenames cross-platform.

### Entry Point

`Sources/swift-symbolicate/SwiftSymbolicate.swift` — `@main` `ParsableCommand` that wires everything together: parses CLI args, reads `SWIFT_BACKTRACE` env for formatting/symbolication options, creates `CrashScanner`, and dispatches to the appropriate reader/writer.

## Package Structure Notes

### Hardcoded Paths in Package.swift

`Package.swift` has hardcoded `unsafeFlags` pointing to a local Swift compiler build at `/Users/carlpeto/Code/swift-project/build/...`. These are needed because the tool uses `@_spi`-annotated APIs from the in-development Swift `Runtime` module that aren't part of a released SDK.

### Package Traits

Traits gate test compilation:
- `TestGeneral` — unit tests that work without running external binaries
- `TestSymbolicating` — integration tests that actually run `crashMe`/`crashMeMultithreaded` and symbolicate
- `DebuggingSymbolicator` — enables verbose logging in the symbolication path
- Default: `TestSymbolicating` + `TestGeneral` both enabled

### Debug Compile Flags

In `Package.swift` (commented out by default):
- `.define("DEBUG_SCANNER")` — verbose byte-level logging in `CrashScanner`
- `.define("DEBUG_RECOGNIZER")` — verbose logging in `Recognizer`

### Test Helpers

`testHelpers.swift` provides:
- `getCrashLog(isJson:isSymbolicated:isMultiThreaded:)` — launches `crashMe` or `crashMeMultithreaded` via `Subprocess` with appropriate `SWIFT_BACKTRACE` env, returns its stderr/stdout
- `testScan(sampleData:process:)` — runs a string through `CrashScanner` with an in-memory stream
- `String.expect(@MatchBuilder)` — result builder DSL for asserting lines match a sequence of strings/regexes in order
